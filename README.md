# Изолят

> Система раннего выявления социальной изоляции школьников на основе теории графов.

Главный пользователь — **школьный психолог**. Ученики **никогда не видят
аналитику** — они только проходят анонимный опрос по коду/QR. Из их ответов
автоматически строится ориентированный социометрический граф, а NetworkX
считает все показатели.

## Как это работает

1. Психолог входит в систему, создаёт **класс** и добавляет **учеников** —
   каждому автоматически выдаётся уникальный **код**.
2. Психолог создаёт **срез** (опрос). Тексты трёх вопросов можно адаптировать
   под возраст класса. Система выдаёт **ссылку + QR-код** и печатный список кодов.
3. Ученик открывает страницу опроса, вводит свой код и отвечает кнопками на три
   вопроса. Повторно пройти опрос нельзя.
4. Граф и метрики (изоляты, взаимные выборы, входящие/исходящие связи,
   сплочённость, мосты, компоненты, degree/betweenness centrality, density,
   сообщества) считаются автоматически и рисуются через **vis-network**.
5. **Time Slider** переключает срезы — граф перестраивается. По клику на узел
   открывается **карточка ученика**: история связей, показатели, динамика,
   заметки, встречи, эффективность вмешательства.

## Технологии

- **Backend:** Python, FastAPI, SQLAlchemy, NetworkX, **PostgreSQL (Supabase)**.
- **Frontend:** HTML, CSS, vanilla JavaScript, vis-network.
- Авторизация: собственные токены (PBKDF2 + HMAC).

## Запуск

Нужен **Python 3.10+** и **проект в Supabase** (база — только Supabase, без SQLite
и без `.env`).

### 1. Строка подключения к Supabase
Supabase → **Project Settings → Database → Connection string → URI**. Скопируй
строку. Если прямое подключение `db.<ref>.supabase.co` не коннектится — возьми
**Session pooler** (порт 5432, работает по IPv4).

### 2. Впиши её в `app/config.py`
Это **единственное**, что нужно отредактировать. Открой `app/config.py` и в
переменную `DATABASE_URL` вставь строку, **заменив схему** на
`postgresql+psycopg2://` и подставив пароль:

```python
DATABASE_URL = "postgresql+psycopg2://postgres:ТВОЙ_ПАРОЛЬ@db.xxxx.supabase.co:5432/postgres"
```

(Спецсимволы в пароле URL-кодируй: `@`→`%40`, `#`→`%23`, `:`→`%3A`.)

### 3. Установка и запуск

```bash
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt

python -m app.seed                 # (необязательно) демо-класс с данными
python run.py                      # или: uvicorn app.main:app --reload --port 8000
```

Открой <http://127.0.0.1:8000>, зарегистрируйся (это аккаунт психолога) и войди.
Таблицы создаются в Supabase автоматически при первом запуске.

Демо-аккаунт (после `python -m app.seed`):

```
Вход:   demo@izolyat.local
Пароль: demo12345
```

## Как пользоваться (полный сценарий)

### Психолог (веб-кабинет)
1. **«+ Класс»** — создай класс.
2. **«Ученики»** — добавь учеников. Каждому присваивается уникальный **код**.
3. **«+ Срез»** — создай опрос (по умолчанию открыт), при желании поменяй тексты
   вопросов под возраст класса.
4. **«🔗 Ссылка и QR»** — здесь ссылка на опрос, QR-код и список кодов.
   Кнопка **«Печать кодов»** — распечатать и раздать.

### Ученик (заполнение формы)
1. Сканирует **QR** или открывает **ссылку** `http://.../survey.html?survey=1`.
2. Вводит свой **код**.
3. Отвечает кнопками на 3 вопроса и жмёт **«Отправить»**.

### Чтобы ученики открыли форму со своих телефонов
Ссылка `127.0.0.1` работает только на том же компьютере. Для реального класса:
- **Задеплой** (например, Render) — публичный URL, QR открывается с любого телефона.
- **Одна Wi-Fi:** запусти `uvicorn app.main:app --host 0.0.0.0 --port 8000` и
  **открой кабинет по IP компьютера** (`http://192.168.x.x:8000`), а не по
  `127.0.0.1` — тогда ссылка/QR будут указывать на этот IP.

### Проверить опрос за 2 минуты (локально)
Создай срез → **«🔗 Ссылка и QR»** → скопируй ссылку и любой код → открой ссылку в
новой вкладке → введи код → ответь. Вернись в кабинет — граф перестроится.

## Приватность

- Ученики не являются пользователями — они физически не могут открыть аналитику.
- Каждый запрос психолога ограничен его `psychologist_id`; ученики ходят только
  на публичные эндпоинты опроса.
- Анонимные коды: ученик отвечает по коду, не раскрывая личность одноклассникам.

## Примечания

- `vis-network` и библиотека QR подключаются с CDN — нужен интернет при открытии
  кабинета психолога. Страница опроса ученика работает без внешних библиотек.
- Метрики графа считаются на положительных выборах (кино + проект); вопрос «кто
  часто остаётся один» агрегируется отдельно как сигнал изоляции.
- Если меняешь схему у уже созданной базы — `create_all` не изменяет
  существующие таблицы. Пересоздай их или выполни нужный `ALTER TABLE` в
  Supabase SQL Editor.
